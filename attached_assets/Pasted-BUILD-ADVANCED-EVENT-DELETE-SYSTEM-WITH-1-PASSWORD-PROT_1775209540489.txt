BUILD ADVANCED EVENT DELETE SYSTEM WITH:
1. PASSWORD PROTECTION
2. SOFT DELETE (ARCHIVE)
3. HARD DELETE (PERMANENT)
4. RESTORE OPTION
5. FULL DATA CASCADE CLEANUP
6. AUDIT LOG TRACKING

---

🎯 GOAL:

Admin should have 2 delete options:
1. Archive Event (Safe)
2. Delete Permanently (Dangerous, password required)

---

🧩 UI CHANGES:

On clicking DELETE icon:

→ Open modal with 2 options:

-----------------------------------
TITLE: "Manage Event Deletion"

MESSAGE:
"Choose an action. Archive is reversible. Permanent delete will erase all data."

BUTTONS:

1. Archive Event (Primary button)
2. Delete Permanently (Red button)
3. Cancel
-----------------------------------

---

🟡 OPTION 1: ARCHIVE (SOFT DELETE)

When clicked:

→ Mark event as:
   status = "archived"

→ Remove from:
   - Upcoming
   - Ongoing
   - Completed

→ Add new tab:
   "Archived"

→ Data should NOT be deleted

---

📦 ARCHIVED TAB FEATURES:

Each archived event should have:

- Restore button
- Permanent Delete button

---

🔁 RESTORE FUNCTION:

On clicking restore:

→ Move event back to correct section:
   IF event date > today → Upcoming
   IF event is today → Ongoing
   IF event date < today → Completed

→ Show success message: "Event restored"

---

🔴 OPTION 2: PERMANENT DELETE (HIGH SECURITY)

When clicked:

→ Open SECOND confirmation modal:

-----------------------------------
TITLE: "Confirm Permanent Deletion"

MESSAGE:
"This will permanently delete event and ALL related data:
attendance, check-in/out, referrals, payments, claims.
This cannot be undone."

INPUT:
- Password field (required)

BUTTONS:
- Cancel
- Delete Permanently (Red)
-----------------------------------

---

🔐 PASSWORD VALIDATION:

IF password empty:
 → show "Password required"

IF password incorrect:
 → show "Incorrect password"
 → stop process

IF password correct:
 → proceed with FULL DELETE

---

🔥 BACKEND: FULL CASCADE DELETE

When permanent delete is confirmed:

DELETE:

1. Event record
2. Attendance (check-in / check-out)
3. Break logs
4. Assigned crew
5. Referral data
6. Payments
7. Event claims
8. Any logs tied to event

IMPORTANT:
→ No orphan records
→ Use CASCADE or manual delete sequence

---

📊 AUDIT LOG SYSTEM (VERY IMPORTANT)

Every action must be logged:

Log fields:
- action_type: "ARCHIVE" / "RESTORE" / "DELETE"
- event_id
- event_name
- admin_email
- timestamp

Store in:
→ "activity_logs" table

---

⚡ SUCCESS STATES:

Archive:
→ "Event archived"

Restore:
→ "Event restored"

Delete:
→ "Event permanently deleted"

→ Auto refresh list
→ Update counts
